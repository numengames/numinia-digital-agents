This guide enhances the understanding and effective utilization of the file structure implemented at **Numen Games**, inspired by the organizational principles of **Numinia**.

It integrates gamification into document processes, creating a dynamic system suitable for both humans and automated management systems.

This guide is continuously evolving. This version includes enhanced features and refinements from previous feedback to improve user experience and compliance.

## File Structure Overview

Our file system is structured around four primary guilds, each representing crucial organizational functions.

### Example file path

```text
/Guilds/Alchemists/2024_05_15-Innovation_Report-v0.2.0.docx
```

This example demonstrates how files should be categorized and named within the **Alchemists Guild**.

## Document Management Protocols

Each document should follow a standardized naming convention specific to its guild to maintain consistency across the organization.

For instance, all documents in the **Exegetes Guild** should start with the date, followed by the document title and version:

```text
2024_05_15-Research_Paper-v0.1.0.docx
```

### Creation and Storage

* **Location example:** `2024_05_11-Meeting_Notes-v0.1.0.docx` should be stored in `Exegetes/Meetings/2024`.
* **File uniqueness:** Use links and shared locations to prevent duplication and enhance system efficiency.

### Updates and Maintenance

* **Revisions:** Semi-annual reviews ensure documents are current and accurate.
* **Versioning example:** Maintain a version log in a format like `v0.1.0`, `v0.2.0`, and so on, to track document revisions.

## File Naming and Folder Distribution Practices

Consider implementing a document lifecycle policy that details the steps for archiving documents and the process for document disposal or archiving when documents reach the end of their active use.

### Naming Conventions

Use formats like the following:

```text
YYYY_MM_DD-Document_Title-vX.X.X
```

Example:

```text
2024_05_15-Policy_Review-v0.2.0.docx
```

#### Version positions

* **Position one: major version (`X.0.0`)**  
  Represents significant changes or milestones in the document's development. A change in this number usually indicates a major revision or release with substantial modifications that may not be backward-compatible.

* **Position two: minor version (`0.X.0`)**  
  Indicates smaller updates or feature additions that are backward-compatible. Changes in this number reflect enhancements or improvements that build upon the previous version without altering its core functionality or structure.

* **Position three: patch version (`0.0.X`)**  
  Used for minor changes, bug fixes, or corrections that do not introduce new features or significant changes. It reflects maintenance updates that ensure the document remains accurate and functional.

#### Naming rules

* Avoid special characters to ensure names are compatible with all systems.
* Use a capital letter at the start of every word.
* Represent spaces with underscores: `_`
* Use hyphens for conceptual separation: `-`

### Folder Organization

* **Hierarchy:** Organize files by theme or project within their respective guild folders.
* **Limit depth:** Maintain a structure with few subfolders to simplify navigation.
* **Consistency:** Maintain uniform naming and structural patterns to facilitate automated systems and AI integration.

## Training and Orientation for New Employees

**Purpose:** To ensure all new employees understand and efficiently utilize the Numen Games file structure system, facilitating quick integration into our organizational processes.

### Training Schedule

* **Monthly onboarding sessions:** New employees participate in comprehensive onboarding sessions at the start of each month. These sessions cover the basics of the file management system, access protocols, and security measures.
* **Quarterly refresher courses:** To keep employees updated on changes to the file system and reinforce best practices, refresher courses are provided every quarter.

### Training Modules

1. **Introduction to Numen Games File Structure**  
   An overview of the file structure, including the purpose of each guild's specific folders and the logic behind categorization.

2. **Document Management Protocols**  
   Detailed guidance on creating, storing, updating, and archiving documents according to organizational standards.

3. **Security and Compliance**  
   Training on security measures, role-based access control, and how to maintain compliance with ISO standards.

4. **Using the Document Management System**  
   Practical instructions on how to navigate the system, including hands-on practice sessions using test environments.

### Assessment and Certification

* **Assessment:** At the end of the training, employees undergo an assessment to test their understanding and practical knowledge of the file management system.
* **Certification:** Employees who pass the assessment receive a certification acknowledging their proficiency in handling the organization's documentation effectively.

### Continuous Improvement

* **Feedback mechanism:** After each training session, participants are encouraged to provide feedback, which is used to improve future training sessions.
* **Ongoing support:** A dedicated helpdesk is available for employees to address any queries or issues they may encounter while using the file management system.
