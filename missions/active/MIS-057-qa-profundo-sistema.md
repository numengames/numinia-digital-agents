## Story

As a team of digital and biological agents, we want to do a deep QA of the complete NWOS system, to ensure everything is coherent, the lifecycle is well implemented, and the system is ready to grow with real business metrics.

---

## QA Areas

### 1. Documentary coherence
- All seminal documents present in canon/ and referenced in INDEX.md
- README.md reflects the real structure
- All agents have complete profiles
- Missions in INDEX, decisions in INDEX

### 2. BOOT/EXECUTE/COMMIT cycle
- P-001 (briefing) reflects the complete cycle
- The session close protocol includes git push of memory
- Agents know exactly what to do in each phase

### 3. Human-in-the-Loop with full context
- Every approval request includes:
  - What will be done exactly
  - Epistemic value (what we will learn)
  - Pragmatic value (what impact it has)
  - Practical example if necessary
  - Approval importance score (1-10)

### 4. Agent architecture
- Evaluate whether agents should be in a top layer outside of guilds
- Guilds as organizational structure vs. agents as system actors

### 5. Business metrics
- Define measurable KPIs for the system
- Cost per mission, execution speed, output quality
- ROI of digital agents

## Epistemic value

A deep QA reveals inconsistencies that can only be seen when the system is observed as a whole. Each gap found is a learning about how a living system accumulates architectural debt.

## Pragmatic value

The system needs to be coherent before incorporating new agents and new organizations. A solid foundation is worth more than 10 features on weak foundations.
